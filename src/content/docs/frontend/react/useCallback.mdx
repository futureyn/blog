#### 本小结介绍 useCallback 的用法和使用场景

如果该组件需要多次渲染那么使用 useCallback 函数 可以缓存该回调函数。让其缓存起来，减少不必要的渲染。


[//]: # (1. 当函数被传递给子组件时，子组件可能会在每次父组件重新渲染时重新创建，这可能会导致子组件重新渲染，导致性能问题。)
[//]: # (2. 当函数被传递给 useEffect 或 useMemo 时，可能会导致无限循环。)

[//]: # (3. 当函数被传递给 useCallback 时，可以避免在每次父组件重新渲染时重新创建函数，从而避免子组件重新渲染。)

[//]: # (4. 当函数被传递给 useMemo 时，可以避免在每次父组件重新渲染时重新计算结果，从而避免子组件重新渲染。)

[//]: # (5. 当函数被传递给 useReducer 时，可以避免在每次父组件重新渲染时重新创建 reducer，从而避免子组件重新渲染。)
